# 4. Regulile de Acces Static vs. Non-Static


### Subiecte
- Contextul Static (al Clasei) vs. Contextul Non-Static (al Instanței)    
- Reguli de Acces între Contextul Static și Non-Static
- Metode de Accesare a Membrilor Statici

### Rezumat
 Acest ghid consolidează diferența fundamentală dintre membrii `static` (care aparțin clasei) și membrii `non-static` (care aparțin unei instanțe/obiect). Pe baza rezultatelor testului, înțelegerea regulilor de acces este solidă. Ghidul detaliază cele trei reguli cheie: de ce un context static nu poate accesa un membru non-static (regula principală), de ce inversul este permis și care este modalitatea corectă de a accesa membrii statici din exteriorul clasei.

### Concepte
- **Regula 1: `static` → `non-static` este INTERZIS.** Un context static (cum ar fi o metodă `public static void main`) nu poate accesa direct un membru non-static (o variabilă de instanță sau o metodă non-statică).
- **Motivul (Concept Cheie):** Contextul static aparține clasei și rulează fără o instanță specifică. Membrii non-statici "trăiesc" doar în interiorul unei instanțe. Deoarece contextul static nu are o referință `this` la o instanță, nu știe _al cui_ membru non-static să îl acceseze. Acest conflict este detectat ca o **eroare de compilare**.
- **Regula 2: `non-static` → `static` este PERMIS.** Un context non-static (o metodă de instanță) _poate_ accesa și modifica membrii statici. Obiectul (instanța) este conștient de membrii statici (partajați) ai clasei din care face parte.
- **Regula 3: Accesarea Membrilor Statici.** Membrii `static` ar trebui accesați întotdeauna folosind numele clasei (ex: `Vehicul.MAX_SPEED`). Deși Java permite accesarea lor printr-o variabilă de instanță (ex: `v.MAX_SPEED`), această practică este puternic descurajată deoarece creează confuzie.
- **Regula 4: Apeluri Între Metode.** O metodă `static` poate apela direct alte metode `static`. O metodă `static` _nu poate_ apela direct metode non-statice (din același motiv ca Regula 1).

### Vocabular
- **`static`:** Un cuvânt cheie care indică faptul că un membru (câmp sau metodă) aparține _clasei_ în sine, nu unei instanțe specifice. Există o singură copie a unui membru static, partajată de toate obiectele.
- **`non-static` (de instanță):** Un membru care aparține fiecărui _obiect_ (instanță) individual. Fiecare obiect creat primește propria sa copie a membrilor non-statici.
- **Context Static:** Un bloc de cod (de obicei o metodă statică) care este executat fără a fi asociat cu o instanță anume.
- **`this`:** O referință (implicită) la instanța _curentă_ a obiectului. `this` este disponibil doar în context non-static (ex: în constructori și metode de instanță) și este motivul pentru care contextul static nu poate accesa membrii non-statici.
